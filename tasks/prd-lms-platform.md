# PRD: LMS Platform (Learning Management System)

## 1. Введение / Обзор

LMS-платформа для онлайн-обучения, предназначенная для студентов, получивших доступ через внешнюю систему оплаты (Telegram-бот). Платформа позволяет студентам проходить видеокурсы, отслеживать прогресс и получать сертификаты. Администраторы управляют контентом и студентами через отдельную панель.

Платформа доступна в двух форматах:
- **Веб-приложение** — полный функционал для студентов и администраторов
- **Telegram Mini App** — полный студенческий функционал (без админ-панели)

---

## 2. Цели

1. Предоставить студентам удобный доступ к видеокурсам через браузер и Telegram
2. Обеспечить автоматическое сохранение прогресса просмотра видео
3. Автоматически выдавать сертификаты при 100% прохождении курса
4. Дать администраторам полный контроль над контентом и студентами
5. Поддержать два языка интерфейса: русский и казахский
6. Масштабироваться на 500+ студентов с самого начала
7. Предоставить внешний API для интеграции с Telegram-ботом (выдача доступа)

---

## 3. Пользовательские истории

### Студент
- Как студент, я хочу войти по email и паролю, чтобы получить доступ к своим курсам
- Как студент, я хочу смотреть видео через удобный плеер и продолжать с того места, где остановился
- Как студент, я хочу видеть свой прогресс по каждому курсу
- Как студент, я хочу скачать сертификат при завершении курса на 100%
- Как студент, я хочу пользоваться платформой в Telegram без выхода из приложения
- Как студент, я хочу переключать язык интерфейса (русский / казахский)

### Администратор
- Как администратор, я хочу создавать курсы и добавлять видео (YouTube-ссылки или загрузка)
- Как администратор, я хочу управлять студентами (добавлять, блокировать, редактировать)
- Как администратор, я хочу видеть статистику активности студентов
- Как администратор, я хочу настраивать шаблоны сертификатов
- Как администратор, я хочу выдавать или отзывать сертификаты вручную

### Telegram-бот (внешняя система)
- Как Telegram-бот, я хочу вызвать API LMS после подтверждения оплаты, чтобы создать аккаунт студента и отправить ему данные для входа

## 4. Функциональные требования

### 4.1 Аутентификация

1. Вход по email + пароль для студентов
2. Отдельный вход для администраторов (роль задаётся в БД)
3. Аутентификация через Telegram `initData` для Telegram Mini App
4. JWT-токены для авторизации API-запросов
5. Защищённые маршруты (редирект неавторизованных пользователей)
6. Функция "Забыли пароль" через email
7. Сессия хранится в httpOnly cookie

### 4.2 Внешний API (интеграция с Telegram-ботом)

8. `POST /api/external/students` — создание студента: принимает `{ name, email, phone, api_key }`, создаёт аккаунт, возвращает сгенерированный пароль
9. API-ключ для аутентификации внешних запросов (настраивается в env)
10. Опциональная отправка приветственного email студенту при создании аккаунта

### 4.3 Студентская часть — Каталог курсов

11. Список всех доступных курсов с обложкой, названием, описанием, прогрессом (%)
12. Фильтр по категории и статусу (в процессе / завершён)
13. Поиск курсов по названию
14. Карточка курса показывает: количество видео, общую длительность, уровень сложности

### 4.4 Студентская часть — Просмотр курса

15. Страница курса со списком всех видео и прогрессом по каждому
16. Индикация завершённых видео (галочка)
17. Кнопка "Продолжить" — переход к последнему незавершённому видео

### 4.5 Студентская часть — Видеоплеер

18. Единый кастомный HLS-плеер для всех видео (загруженных и YouTube)
19. YouTube-видео: скачиваются через yt-dlp, конвертируются в HLS через FFmpeg, хранятся в Supabase Storage
20. Загруженные видео: конвертируются в HLS через FFmpeg, хранятся в Supabase Storage
21. Функции плеера: Play/Pause, регулировка скорости (0.5x, 1x, 1.25x, 1.5x, 2x), перемотка ±10 сек, полноэкранный режим, Mute/Volume, timeline
22. Адаптивный bitrate (480p, 720p, 1080p) если доступно
23. Автосохранение прогресса каждые 10 секунд
24. Восстановление воспроизведения с сохранённого места при повторном открытии
25. Видео считается завершённым при просмотре >90%
26. Кнопки навигации "Предыдущее видео" / "Следующее видео"

### 4.6 Студентская часть — Профиль

27. Просмотр и редактирование: имя, фамилия, телефон, аватар, биография
28. Email не редактируется студентом
29. Смена пароля
30. Статистика: общее время обучения, количество завершённых курсов, общий процент прогресса

### 4.7 Сертификаты

31. Автоматическая выдача сертификата при достижении 100% прохождения курса
32. Ручная выдача или отзыв сертификата администратором
33. PDF-генерация сертификата: имя студента, название курса, дата выдачи, уникальный номер, QR-код
34. Страница сертификатов студента со списком завершённых курсов
35. Кнопка "Скачать PDF"
36. Публичная страница верификации сертификата по QR-коду (без авторизации)

### 4.8 Многоязычность

37. Полный перевод интерфейса на русский и казахский языки
38. Автоопределение языка: из настроек браузера или Telegram
39. Ручное переключение языка в шапке (кнопка ru / kz)
40. Выбор языка сохраняется в localStorage и профиле пользователя

### 4.9 Telegram Mini App

41. Та же Next.js страница, открывается в Telegram как Mini App
42. Аутентификация через Telegram `initData` (верифицируется на сервере)
43. Если аккаунт не найден — предложение войти по email
44. Доступно: каталог курсов, просмотр видео, прогресс, профиль, сертификаты
45. Недоступно: админ-панель
46. UI адаптирован под мобильный Telegram: bottom navigation, touch-friendly кнопки

### 4.10 Админ-панель — Управление курсами

47. Список всех курсов с кнопками создать / редактировать / удалить
48. Создание курса: название (ru/kz), описание (ru/kz), категория, уровень сложности, обложка, порядок сортировки, статус (черновик / опубликован)
49. Управление видео в курсе: добавить YouTube-ссылку или загрузить файл, редактировать название/описание, drag-and-drop для изменения порядка, удалить
50. Статус обработки видео (в очереди / обрабатывается / готово / ошибка)
51. Предпросмотр курса глазами студента

### 4.11 Админ-панель — Управление студентами

52. Таблица студентов: имя, email, телефон, дата регистрации, статус, прогресс
53. Поиск по имени / email
54. Фильтры: по статусу (активен / заблокирован), по прогрессу
55. Просмотр детального профиля студента: прогресс по каждому курсу, история активности
56. Редактирование данных студента администратором
57. Блокировка / разблокировка доступа
58. Сброс прогресса студента по курсу
59. Ручное создание студента из админки
60. Экспорт списка студентов в CSV

### 4.12 Админ-панель — Статистика (Дашборд)

61. KPI-карточки: всего студентов, активных за 7 дней, завершённых курсов, выданных сертификатов
62. График активности студентов по дням (последние 30 дней)
63. Топ-5 популярных курсов по числу студентов
64. Таблица студентов с сортировкой по прогрессу

### 4.13 Админ-панель — Сертификаты

65. Список всех выданных сертификатов с фильтрами
66. Ручная выдача сертификата студенту
67. Отзыв сертификата (с указанием причины)
68. Настройка шаблона: загрузка логотипа, текст подписи

## 7. Технические соображения

### Стек
- **Frontend + API:** Next.js 14+ (App Router), TypeScript, Tailwind CSS, Shadcn/ui
- **База данных + Auth + Storage:** Supabase (PostgreSQL + Auth + Storage)
- **i18n:** next-intl (русский + казахский)
- **Видеоплеер:** HLS.js с кастомными контролами
- **Обработка видео:** Отдельный Node.js воркер с yt-dlp + FFmpeg + Bull (очередь задач)
- **PDF:** PDFKit + qrcode
- **Telegram Mini App:** @telegram-apps/sdk

### Аутентификация
- Студенты/Админы: Supabase Auth (JWT)
- Telegram Mini App: верификация `initData` через HMAC-SHA256 на сервере, затем выдача Supabase сессии
- Внешний API (Telegram-бот): статический API-ключ в заголовке `X-API-Key`

### Видео-воркер
- Отдельный Node.js сервис на том же VPS (Docker container)
- Принимает задачи через Bull Queue (Redis, развёрнутый на VPS)
- YouTube: `yt-dlp url -o output.mp4` → FFmpeg → HLS сегменты → Supabase Storage (self-hosted)
- Загрузка: FFmpeg → HLS сегменты → Supabase Storage (self-hosted)
- Уведомляет Next.js API об успехе/ошибке через webhook

### Деплой — всё на одном VPS
- **Self-hosted Supabase** через официальный Docker Compose (PostgreSQL, Auth, Storage, Realtime, Kong)
- **Next.js** — Docker container, PM2 или напрямую через Node.js
- **Видео-воркер** — отдельный Docker container на том же VPS
- **Redis** — Docker container для Bull Queue
- **Nginx** — reverse proxy: раздаёт Next.js, Supabase API, HLS-видео
- **SSL** — Let's Encrypt через Certbot или Nginx Proxy Manager
- **Docker Compose** — оркестрация всех сервисов единым файлом

### Масштабируемость
- При росте нагрузки — перенос видео-воркера на отдельный VPS
- Supabase Storage хранит видео локально на VPS (настраивается путь)
- Nginx настраивается как CDN-прокси при необходимости

## 9. Открытые вопросы

1. **Email-уведомления:** Нужны ли автоматические email при выдаче сертификата или напоминания о незавершённых курсах?
2. **Видео-лимиты:** Максимальный размер загружаемого видео? (рекомендуется 5 ГБ)
3. **VPS характеристики:** Какой сервер планируется? (RAM, CPU, диск) — влияет на производительность видео-обработки и хранения
4. **Домен:** `demo-lms.aibot.kz` — основной домен платформы. Supabase API будет на `demo-lms.aibot.kz/supabase` или на поддомене `supabase.aibot.kz` (настраивается через Nginx).
5. **yt-dlp риск:** YouTube может заблокировать скачивание — нужен ли fallback вариант (YouTube iframe с кастомным UI)?
6. **Резервное копирование:** Нужна ли стратегия бэкапа БД и видео на внешнее хранилище (S3, Backblaze B2)?
7. **Self-hosted Supabase:** Потребует ~2–4 ГБ RAM только для своих сервисов. Остальные сервисы (Next.js, воркер, Redis) добавят ещё ~1–2 ГБ. Рекомендуется VPS от 8 ГБ RAM.
